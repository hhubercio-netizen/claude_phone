### 1.2 What changes in this sub-spec

- Add `tower-governor` dependency to enable per-IP HTTP-layer rate limiting (`TM-RATE.1`).
- Add an in-process auth-failure tracker keyed by `IpAddr` with sliding-window count + exponential backoff (`TM-RATE.2`).
- Add a per-connection sliding-window message rate cap inside `phone_ws.rs` and `wrapper_ws.rs` receive loops (`TM-RATE.3`).
- Wrap every `sink.send(...)` in both WS handlers with `tokio::time::timeout` so a stalled writer drops the connection rather than parking forever (`TM-RATE.6`).
- Track `Message::Pong` arrivals, drop the connection if no pong is received within 90 s of the last pong (3× the 30 s ping interval — two missed beats before kill) (`TM-RATE.7`).
- Configure hyper's `http1_header_read_timeout` to 10 s on the `axum::serve` builder so a slow-loris on the upgrade headers cannot pin a FD (`TM-RATE.9`).
- Reconcile threat-model row `TM-RATE.4` (8 MB → 64 KB) — see §2.

## 2. Threat-model reconciliation: TM-RATE.4 cap

**Catalog row today** (`2026-05-23-threat-model.md:643`):
```
| TM-RATE.4  | Per-session memory hard cap 8 MB; drop oldest on overflow | TODO |
```

**Code today** (`crates/claude-phone-gateway/src/session/session.rs:24`):
```rust
pub const PHONE_BUFFER_BYTES_CAP: usize = 64 * 1024;
```

The 8 MB figure in the catalog originated from the initial DoS section draft and was never updated when `PHONE_BUFFER_BYTES_CAP` was chosen empirically against typical xterm scrollback (~600 lines × ~100 cols ≈ 64 KB). The code is correct and conservative — 64 KB drops oldest binary frames as soon as the cap is hit, which is exactly the policy the catalog describes minus the wrong number.

**Decision:** keep the code at 64 KB, correct the catalog to match. Doc-only diff, no code change. Status flips from `TODO` to `GREEN`. Rationale, in one sentence inside the row, points the reader at `session.rs:24` so future audits don't re-open the gap.

`PHONE_BUFFER_FRAMES_CAP = 256` (`session.rs:25`) is the secondary cap and is already documented in the comment block above it. No catalog change for the frame count.

## 3. Dependency decision: tower-governor

**Add:** `tower-governor = "0.5"` (Apache-2.0, depends on `governor = "0.7"`, `tower = "0.5"`, `tracing = "0.1"`; no transitive native deps).

**Why a crate, not in-house:**

- `governor` (the underlying library) implements a **generic cell rate limiter** with a leaky bucket, lock-free atomics, and well-tested overflow/wraparound semantics. Reimplementing this correctly under contention is a multi-week project that nobody in this repo is paid to maintain.
- The tower wrapper handles the awkward bits of WS upgrades: when a route is `any(...)` and the inner future swaps protocols, naive middleware double-counts the connection. `tower-governor` already special-cases this via its `peer_ip_extractor` model.
- Maintenance load is small: 1 dep, 13 transitive crates, audited via `cargo audit` in `.github/workflows/ci.yml` (which 4.7 already pins).

**Alternatives weighed:**

1. **In-house token bucket on `DashMap<IpAddr, BucketState>`.** Rejected — reimplements a battle-tested primitive, adds 200+ LOC under contention, and is a year of unmaintained surface area waiting to bite us.
2. **`tower-limit` (mainline tower).** Rejected — only does concurrency limits, no rate per second. We need both.
3. **Nginx / Cloudflare-only.** Rejected — defense in depth requires the origin to enforce a floor even if CF is bypassed (direct-IP scan, internal traffic, dev environments).

**Per-connection rate (TM-RATE.3) does NOT need a new dependency.** Inside each WS receive loop we maintain a `VecDeque<Instant>` of timestamps for the last N frames (N = 100 for phone→gw, 1000 for gw→phone) and reject on overflow within a 1 s window. Sliding window, ~15 LOC per loop. This avoids pulling `governor` into the WS task scope.

**Locked-in answer for OQ-4** (`security-hardening-design.md` §13 OQ-4): sliding window via `tower-governor` leaky bucket for HTTP layer; sliding window via in-process `VecDeque<Instant>` for per-connection layer. Both are sliding windows; both reject (drop the frame / close the upgrade) rather than queue.

### 4.2 New module: `crates/claude-phone-gateway/src/rate_limit.rs`

This module owns the per-IP HTTP-layer limiter and the auth-attempt tracker. Two responsibilities, one module — they share IP-keyed lookup primitives and live closer together than they would as one-off helpers inside `http.rs` and `wrapper_ws.rs`.

```rust
//! Per-IP rate limiters wired into the HTTP layer and WS auth path.
//!
//! Sliding window via governor's leaky-bucket cell rate limiter for raw
//! connection rate (TM-RATE.1). Separate in-process counter for failed
//! WrapperHello auth attempts (TM-RATE.2) so we can apply exponential
//! backoff specifically to auth failures, not generic request rate.

use std::net::IpAddr;
use std::num::NonZeroU32;
use std::sync::Arc;
use std::time::{Duration, Instant};

use dashmap::DashMap;
use governor::{
    clock::DefaultClock,
    state::{InMemoryState, NotKeyed},
    Quota, RateLimiter,
};

// --- TM-RATE.1: per-IP HTTP-layer cap ---------------------------------------

/// 5 requests per second per IP, burst of 10. Sliding window via leaky
/// bucket. WS upgrade is one request — sustained 5/s lets a legit user
/// reconnect on flaky networks without tripping the limiter, while
/// blocking flood-style attackers at the first burst.
pub const PER_IP_REQ_PER_SEC: u32 = 5;
pub const PER_IP_BURST: u32 = 10;

// --- TM-RATE.2: per-IP auth-failure tracker --------------------------------

/// Cap on failed WrapperHello auths per IP within `AUTH_FAIL_WINDOW`.
pub const AUTH_FAIL_THRESHOLD: u32 = 10;
pub const AUTH_FAIL_WINDOW: Duration = Duration::from_secs(60);
/// Backoff applied after `AUTH_FAIL_THRESHOLD` failures within window.
/// Exponential: 2^n seconds for n consecutive offenses, capped at 1 h.
pub const AUTH_BACKOFF_BASE_SECS: u64 = 2;
pub const AUTH_BACKOFF_CAP_SECS: u64 = 3600;

#[derive(Debug, Default)]
struct AuthState {
    /// Timestamps of failed attempts in the current window (oldest first).
    failures: std::collections::VecDeque<Instant>,
    /// Count of consecutive backoff escalations (resets on a successful auth).
    escalations: u32,
    /// If Some, IP is locked out until this instant.
    locked_until: Option<Instant>,
}

/// Process-wide auth-failure tracker. Cloneable Arc'd state so handlers
/// share the same view.
#[derive(Clone)]
pub struct AuthRateLimiter {
    inner: Arc<DashMap<IpAddr, AuthState>>,
}

impl AuthRateLimiter {
    pub fn new() -> Self {
        Self { inner: Arc::new(DashMap::new()) }
    }

    /// Returns `true` if this IP is currently locked out. Side effect: prunes
    /// failures outside the current window so the entry stays bounded.
    pub fn is_locked(&self, ip: IpAddr) -> bool {
        let now = Instant::now();
        if let Some(state) = self.inner.get(&ip) {
            if let Some(until) = state.locked_until {
                return now < until;
            }
        }
        false
    }

    /// Record a failed WrapperHello auth. If the threshold is breached within
    /// the window, lock the IP for an exponentially-growing backoff.
    pub fn record_failure(&self, ip: IpAddr) {
        let now = Instant::now();
        let mut entry = self.inner.entry(ip).or_default();
        let cutoff = now - AUTH_FAIL_WINDOW;
        while entry.failures.front().is_some_and(|t| *t < cutoff) {
            entry.failures.pop_front();
        }
        entry.failures.push_back(now);
        if entry.failures.len() as u32 >= AUTH_FAIL_THRESHOLD {
            entry.escalations = entry.escalations.saturating_add(1);
            let secs = AUTH_BACKOFF_BASE_SECS
                .saturating_pow(entry.escalations)
                .min(AUTH_BACKOFF_CAP_SECS);
            entry.locked_until = Some(now + Duration::from_secs(secs));
            entry.failures.clear();
        }
    }

    /// Record a successful auth. Resets escalation count so a previously-
    /// throttled-but-now-legitimate operator does not get re-locked by stale
    /// state.
    pub fn record_success(&self, ip: IpAddr) {
        if let Some(mut entry) = self.inner.get_mut(&ip) {
            entry.failures.clear();
            entry.escalations = 0;
            entry.locked_until = None;
        }
    }
}

impl Default for AuthRateLimiter {
    fn default() -> Self {
        Self::new()
    }
}

// --- TM-RATE.3: per-connection message rate limiter -------------------------

/// Sliding-window message rate limiter for inside-WS receive loops. Holds the
/// last `cap` arrival timestamps; the next message is rejected if the oldest
/// timestamp is less than 1 s old.
#[derive(Debug)]
pub struct ConnRateLimiter {
    window: std::collections::VecDeque<Instant>,
    cap: usize,
}

impl ConnRateLimiter {
    pub fn new(cap: usize) -> Self {
        Self { window: std::collections::VecDeque::with_capacity(cap), cap }
    }

    /// Returns `true` if the new arrival is within rate. Always pushes the
    /// timestamp — the caller has already received the frame; we just want to
    /// know whether to keep going or drop the connection.
    pub fn check(&mut self, now: Instant) -> bool {
        if self.window.len() == self.cap {
            if let Some(&oldest) = self.window.front() {
                if now.duration_since(oldest) < Duration::from_secs(1) {
                    return false;
                }
            }
            self.window.pop_front();
        }
        self.window.push_back(now);
        true
    }
}

/// 100 msg/s cap for phone → gateway direction (mostly keystrokes — generous).
pub const PHONE_TO_GW_MSG_PER_SEC: usize = 100;
/// 1000 msg/s cap for gateway → phone direction (PTY bursts — needs headroom
/// for `claude` rendering a large response).
pub const GW_TO_PHONE_MSG_PER_SEC: usize = 1000;
```

`// TM-RATE.1` `// TM-RATE.2` `// TM-RATE.3` anchor comments are placed above the constants and the `is_locked` / `record_failure` / `check` methods so future grep against the catalog hits them directly.

`rate_limit.rs` is exposed from `lib.rs`:

```rust

```

### 4.3 `crates/claude-phone-gateway/src/http.rs` — wire GovernorLayer

Add the per-IP HTTP-layer rate limiter (`TM-RATE.1`) and propagate the `AuthRateLimiter` into `WrapperWsState`.

```rust
use std::net::SocketAddr;

use tower_governor::{governor::GovernorConfigBuilder, GovernorLayer};

use crate::rate_limit::{AuthRateLimiter, PER_IP_BURST, PER_IP_REQ_PER_SEC};

pub fn build_app(config: &GatewayConfig) -> anyhow::Result<Router> {
    // ... existing registry / sweeper / state code unchanged ...

    let auth_rate_limiter = AuthRateLimiter::new();

    let wrapper_state = wrapper_ws::WrapperWsState {
        registry: registry.clone(),
        allowed_keys,
        public_origin: config.public_origin.clone(),
        auth_rate_limiter: auth_rate_limiter.clone(),
    };

    // ... existing phone_state / static_state / assets / trace_layer /
    //     security_headers unchanged ...

    // TM-RATE.1 — per-IP HTTP cap. burst=10 absorbs reconnect storms on
    // flaky mobile networks; sustained 5/s blocks WS-flood attackers.
    let governor_conf = Arc::new(
        GovernorConfigBuilder::default()
            .per_second(PER_IP_REQ_PER_SEC as u64)
            .burst_size(PER_IP_BURST)
            .finish()
            .expect("governor config valid at compile time"),
    );

    let app = Router::new()
        .route("/healthz", get(health::handler))
        .route("/api/wrapper", any(wrapper_ws::handler))
        .with_state(wrapper_state)
        .route("/api/phone/:token", any(phone_ws::handler))
        .with_state(phone_state)
        .nest_service("/assets", assets)
        .route("/s/:token", get(statics::session_shell))
        .route("/", get(statics::root))
        .with_state(static_state)
        .layer(GovernorLayer { config: governor_conf })
        .layer(security_headers)
        .layer(CompressionLayer::new())
        .layer(trace_layer);

    Ok(app)
}
```

`GovernorLayer` is added **outside** the security-headers layer so 429 Too Many Requests responses still carry the strict CSP / HSTS / X-Frame-Options headers — defense-in-depth on error pages.

`/healthz` is intentionally inside the layer too. The cost of a single rejected health check is zero (curl loops should not exceed 5/s anyway); the cost of leaving it outside is that a hostile actor can flood `/healthz` to identify the gateway and bypass the limiter for fingerprinting.

### 4.4 `crates/claude-phone-gateway/src/main.rs` — HTTP upgrade timeout

```rust
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let cli = Cli::parse();
    let config = GatewayConfig::load(&cli.config)
        .with_context(|| format!("loading config {:?}", cli.config))?;

    logging::init(config.log_format);
    tracing::info!(bind = %config.bind_addr, "starting gateway");

    let app = build_app(&config)?;
    let listener = tokio::net::TcpListener::bind(config.bind_addr).await?;

    // TM-RATE.9 — bound the slow-loris attack on the HTTP header read phase.
    // hyper's default header_read_timeout is None which lets a hostile peer
    // hold a FD open forever by dribbling header bytes. 10 s is well above
    // any legitimate handshake latency and matches the wrapper-side
    // HELLO_TIMEOUT for symmetry (TM-LEAK.3 sweep checks both).
    axum::serve(listener, app.into_make_service_with_connect_info::<SocketAddr>())
        .with_graceful_shutdown(shutdown_signal())
        .into_future_with_builder(|builder| {
            builder
                .http1()
                .header_read_timeout(Some(std::time::Duration::from_secs(10)));
        })
        .await?;
    Ok(())
}
```

Two changes here:
1. `.into_make_service_with_connect_info::<SocketAddr>()` — replaces the bare `app` so `tower_governor` and the per-handler `ConnectInfo` extractor (used in 4.5 `wrapper_ws` for the auth-failure log) can read the peer IP. This is a **strictly broader** service type, no caller break.
2. `header_read_timeout(Some(10 s))` — the hyper builder hook. If axum 0.7's exact API differs (the symbol name may be `with_hyper_builder`, `tcp_nodelay`, etc.), the plan step will figure out the exact path; spec-level intent is fixed.

> **Implementation note for the plan agent:** axum 0.7's `Serve` struct exposes `.tcp_nodelay()` and `.into_future()`; the hyper builder may need to be configured by switching to `hyper_util::server::conn::auto::Builder` directly and looping over `listener.accept()`. If that path is required, the loop is ~30 LOC and is itself worth a small follow-up code review — but it is a routine pattern in axum-using projects and not novel risk.

### 4.5 `crates/claude-phone-gateway/src/routes/wrapper_ws.rs` — auth-failure tracker, per-conn rate, slow-write timeout, no-pong drop

Five additions, each anchored to a TM ID:

```rust
use std::net::SocketAddr;
use std::sync::atomic::{AtomicU64, Ordering};
use std::sync::Arc;
use std::time::Instant;

use axum::extract::ConnectInfo;

use crate::rate_limit::{
    AuthRateLimiter, ConnRateLimiter, PHONE_TO_GW_MSG_PER_SEC,
};

#[derive(Clone)]
pub struct WrapperWsState {
    pub registry: Arc<SessionRegistry>,
    pub allowed_keys: Arc<Vec<claude_phone_shared::ApiKey>>,
    pub public_origin: Option<String>,
    pub auth_rate_limiter: AuthRateLimiter,   // NEW — TM-RATE.2
}

/// Per-send timeout for the sink. 5 s is generous for a 64 KB frame on any
/// non-pathological link — anything slower is a stalled or hostile peer.
const SINK_SEND_TIMEOUT: std::time::Duration = std::time::Duration::from_secs(5);

/// Grace window after the last pong before the connection is torn down.
/// 90 s = 3× ping interval; one missed pong is normal jitter, two is
/// either a network partition or a peer with a zero-window stall.
const PONG_DEADLINE: std::time::Duration = std::time::Duration::from_secs(90);

pub async fn handler(
    ws: WebSocketUpgrade,
    headers: HeaderMap,
    ConnectInfo(peer): ConnectInfo<SocketAddr>,   // NEW
    State(state): State<WrapperWsState>,
) -> Response {
    // TM-RATE.2 — refuse upgrade if this IP is locked out from prior
    // auth-failure burst. 429 is the closest match in HTTP semantics; we
    // could also send 403 but 429 communicates "try again later" which is
    // the truth.
    if state.auth_rate_limiter.is_locked(peer.ip()) {
        return StatusCode::TOO_MANY_REQUESTS.into_response();
    }

    if let Some(expected) = state.public_origin.as_deref() {
        if let Some(origin) = headers.get(header::ORIGIN).and_then(|v| v.to_str().ok()) {
            if origin != expected {
                return StatusCode::FORBIDDEN.into_response();
            }
        }
    }

    ws.max_message_size(MAX_WS_MESSAGE_BYTES)
        .max_frame_size(MAX_WS_MESSAGE_BYTES)
        .on_upgrade(move |socket| handle_socket(socket, state, peer))
}

async fn handle_socket(
    mut socket: WebSocket,
    state: WrapperWsState,
    peer: SocketAddr,
) {
    let hello = match recv_hello(&mut socket).await {
        Some(h) => h,
        None => return,
    };

    let (api_key, token) = match hello {
        ControlMessage::WrapperHello(h) => (h.api_key, h.token),
        _ => {
            send_error(&mut socket, ErrorCode::ProtocolViolation,
                "expected wrapper_hello".into()).await;
            return;
        }
    };

    if !verify_api_key(&api_key, &state.allowed_keys) {
        // TM-RATE.2 — record the failure BEFORE responding so a brute-forcer
        // hitting in rapid succession ratchets the counter on every miss.
        state.auth_rate_limiter.record_failure(peer.ip());
        // 4.2 sub-spec also emits a structured auth-failure log here; that
        // entry should NOT duplicate — keep the 4.2 emit, this is just the
        // counter side-effect.
        send_error(&mut socket, ErrorCode::InvalidApiKey,
            "unknown api key".into()).await;
        return;
    }
    state.auth_rate_limiter.record_success(peer.ip());

    // ... existing register_wrapper / server_hello / tracing::info! unchanged ...

    let (mut sink, mut stream) = socket.split();
    let mut rx_from_phone = handle.rx;
    let session = handle.session.clone();
    let last_pong = Arc::new(AtomicU64::new(now_epoch_secs()));

    let session_outgoing = session.clone();
    let cancel_outgoing = session.cancel.clone();
    let last_pong_out = last_pong.clone();
    let outgoing_task = tokio::spawn(async move {
        // TM-RATE.3 — phone→gw direction.
        let mut conn_rate = ConnRateLimiter::new(PHONE_TO_GW_MSG_PER_SEC);
        loop {
            let cancelled = cancel_outgoing.cancelled();
            tokio::pin!(cancelled);
            tokio::select! {
                biased;
                _ = &mut cancelled => break,
                msg = stream.next() => {
                    let Some(Ok(msg)) = msg else { break };
                    let frame = match msg {
                        Message::Binary(b) => Frame::Binary(b),
                        Message::Text(t) => Frame::Text(t),
                        Message::Close(_) => break,
                        // TM-RATE.7 — pong path: update timestamp, do not
                        // forward to the peer.
                        Message::Pong(_) => {
                            last_pong_out.store(now_epoch_secs(), Ordering::Relaxed);
                            continue;
                        }
                        _ => continue,
                    };
                    // TM-RATE.3 — drop the connection if the peer floods us.
                    if !conn_rate.check(Instant::now()) {
                        tracing::warn!(
                            session_id = %session_outgoing.id,
                            "wrapper exceeded {} msg/s — closing",
                            PHONE_TO_GW_MSG_PER_SEC,
                        );
                        break;
                    }
                    // ... existing snapshot-sender-and-forward unchanged ...
                }
            }
        }
    });

    let cancel_incoming = session.cancel.clone();
    let last_pong_in = last_pong.clone();
    let incoming_task = tokio::spawn(async move {
        let mut keepalive = tokio::time::interval(std::time::Duration::from_secs(30));
        keepalive.tick().await;
        loop {
            let cancelled = cancel_incoming.cancelled();
            tokio::pin!(cancelled);
            tokio::select! {
                biased;
                _ = &mut cancelled => break,
                frame = rx_from_phone.recv() => {
                    let Some(frame) = frame else { break };
                    let msg = match frame {
                        Frame::Binary(b) => Message::Binary(b),
                        Frame::Text(t) => Message::Text(t),
                    };
                    // TM-RATE.6 — slow-write timeout.
                    match tokio::time::timeout(SINK_SEND_TIMEOUT, sink.send(msg)).await {
                        Err(_) => {
                            tracing::warn!("wrapper sink stalled — closing");
                            break;
                        }
                        Ok(Err(_)) => break,
                        Ok(Ok(())) => {}
                    }
                }
                _ = keepalive.tick() => {
                    // TM-RATE.7 — bail out if pong watchdog has elapsed.
                    let last = last_pong_in.load(Ordering::Relaxed);
                    let elapsed = now_epoch_secs().saturating_sub(last);
                    if elapsed > PONG_DEADLINE.as_secs() {
                        tracing::warn!(elapsed, "wrapper no pong — closing");
                        break;
                    }
                    if tokio::time::timeout(SINK_SEND_TIMEOUT,
                        sink.send(Message::Ping(Vec::new()))).await
                        .map(|r| r.is_err()).unwrap_or(true) {
                        break;
                    }
                }
            }
        }
    });

    // ... existing join! / registry.remove / tracing unchanged ...
}

/// Epoch seconds for cheap atomic comparison in the pong watchdog.
fn now_epoch_secs() -> u64 {
    std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .map(|d| d.as_secs())
        .unwrap_or(0)
}
```

### 4.7 `crates/claude-phone-gateway/src/session/session.rs` — TM-RATE.4 anchor

```rust
/// Cap on the ring-buffer of binary frames held while no phone is attached.
/// Sized for terminal scrollback (~hundreds of lines of typical xterm output).
/// Oldest entries are evicted when the cap is hit — terminals always show the
/// most recent output anyway.
//
// TM-RATE.4 — per-session memory hard cap. The eviction policy in
// `push_buffered` (below) drops oldest binary frames as soon as either

pub const PHONE_BUFFER_BYTES_CAP: usize = 64 * 1024;
pub const PHONE_BUFFER_FRAMES_CAP: usize = 256;
```
