### 1.1 Out of scope here

- TM-WS.3 (fail-closed on missing Origin) — owned by sub-spec 4.13.
- TM-WS.10 (HTTP upgrade-phase timeout) — owned by sub-spec 4.13.
- TM-WS.11 (strict token length on phone WS path) — already GREEN at

- OSC 0/1/2 (window title) — `Accepted risk` per threat model §6.3 B3
  (low impact; window title only). Filter passes these through unchanged.
- Output encoding on the wrapper→phone direction — the wrapper is **inside**
  the trust boundary (the phone IS the user). Filtering bytes claude
  legitimately emits would break terminal rendering. Threat covered by
  TM-INPUT.4 (xterm.js handler audit) on the phone side instead.
- Wrapper→phone TLS-leg byte filtering — out of scope per asset model §3.1
  (claude content is High-sensitivity but its rendering is the feature).

- **OQ-9** — OSC/DCS filter location: **gateway phone→wrapper path**.
  Implementation in §3.3 (new module `input_filter`).

### 1.3 Defense-in-depth posture

Two layers cover phone→wrapper terminal-escape abuse:

1. **Gateway filter** (authoritative) — strips OSC 52/8, drops DCS/APC/PM/SOS
   before bytes reach the wrapper PTY. Stateful so multi-frame splits are
   caught.
2. **xterm.js no-op handlers** (client side) — explicitly registers handlers
   for OSC 52 and OSC 8 that return `true` (consume without action). Defends
   against a hypothetical wrapper compromise sending crafted OSC sequences
   to the phone for the user to read; though wrapper→phone is unfiltered by
   design (claude content), explicit handlers stop xterm.js from interpreting
   adversarial sequences if claude itself ever proxies them through.

Either layer alone closes the user-visible attack. Both together survive
single-layer regression (master spec §1.1 defense in depth).

## 2. Dependencies introduced

**None.** The ANSI input filter is in-house (~120 LOC). The alternative —
adopting `vte` (the same parser alacritty uses) — was considered and
rejected for this use case:

| Option                | Pros                                                  | Cons                                                                                                                 | Decision |
|-----------------------|-------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------|----------|
| In-house state machine| Zero new deps; ~120 LOC; we only need filter not interpret | Must hand-roll multi-frame cross-call state; one more place to maintain                                              | **Adopt** |
| `vte` crate           | Battle-tested parser used by alacritty + many others  | Brings ~5 KLOC of dep surface; designed for interpretation (callbacks), awkward for raw-byte filter; not a hot path  | Reject   |

JSON depth is the `serde_json` default (128 recursion levels). No
`unbounded_depth` feature is enabled anywhere in the workspace; sub-spec
4.14 (code audit) adds a `cargo-deny` rule forbidding it. The default
suffices — the spec ships only a forward-looking test.

Wrapper CLI validation reuses existing `clap` — no new crate.

xterm.js OSC handler registration uses the existing
`term.parser.registerOscHandler` API (already exposed because
`allowProposedApi: true` is set at `useTerminal.ts:67`). No new npm package.

### 2.1 No web/npm changes beyond `useTerminal.ts`

A single file in `web/src/components/Terminal/useTerminal.ts` is touched.

## 3. Detailed changes per file

### 3.1 New file: `crates/claude-phone-gateway/src/input_filter.rs`

Stateful byte-stream filter for the phone→wrapper direction. The design
goal is **fail-safe**: when in doubt, drop the suspect bytes; any
legitimate phone client (xterm.js) never emits OSC / DCS / APC / PM / SOS
in keyboard-input direction (verified empirically by reading
`@xterm/xterm` parser source — `onData` fires on cooked keystrokes and
paste, not on parsed escape sequences).

```rust
//! ANSI escape-sequence filter for the phone→wrapper byte direction.
//!
//! Threat model §6.3 B1-B4: a malicious phone client (actor A4 — insider
//! with valid api_key) could send crafted bytes that the wrapper-side PTY
//! would interpret as:
//!
//! - OSC 52 (TM-INPUT.1): clipboard write — poisons the dev-machine
//!   clipboard so the next paste pivots to a host command.
//! - OSC 8  (TM-INPUT.2): hyperlinks — phishes future readers of the
//!   wrapper's terminal output (operators, logs, etc.).
//! - DCS / APC / PM / SOS (TM-INPUT.3): arbitrary device-control /
//!   private-message sequences whose semantics depend on the terminal
//!   emulator. Not part of any legitimate phone-input traffic.
//!
//! OSC 0/1/2 (window title — threat model §6.3 B3 Accepted risk) passes
//! through unchanged.
//!
//! `AnsiInputFilter` is **stateful across calls** so a sequence split
//! across WebSocket frames (e.g. `ESC ]52;c;` in frame 1, `<base64>BEL`
//! in frame 2) still gets caught.

const OSC_TYPE_DIGITS_CAP: usize = 6;       // no legit OSC type has 7+ digits
const STATE_INPUT_HARD_CAP: usize = 64 * 1024;  // matches MAX_WS_MESSAGE_BYTES

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
enum State {
    Normal,
    SeenEsc,        // saw 0x1B, deciding what comes next
    OscPrefix,      // inside `ESC ]`, reading leading digits until ';'
    OscAllow,       // OSC type known and allowed — stream bytes through
    OscAllowEsc,    // inside OscAllow, just saw ESC (possible ST start)
    OscDrop,        // OSC type is 52 or 8 — drop until terminator
    OscDropEsc,     // inside OscDrop, just saw ESC
    StringDrop,     // inside DCS / APC / PM / SOS — drop until terminator
    StringDropEsc,  // inside StringDrop, just saw ESC
}

pub struct AnsiInputFilter {
    state: State,
    /// Accumulator for the leading digits of an OSC type identifier.
    /// Bounded by OSC_TYPE_DIGITS_CAP — anything longer is treated as
    /// malformed and dropped wholesale.
    osc_type_digits: Vec<u8>,
}

impl Default for AnsiInputFilter {
    fn default() -> Self { Self::new() }
}

impl AnsiInputFilter {
    pub fn new() -> Self {
        Self {
            state: State::Normal,
            osc_type_digits: Vec::with_capacity(OSC_TYPE_DIGITS_CAP),
        }
    }

    /// Feed `input`; return filtered output. Subsequent calls continue
    /// mid-sequence if the previous call ended inside one. Returns an
    /// empty Vec when every byte of `input` was inside a dropped sequence.
    pub fn push(&mut self, input: &[u8]) -> Vec<u8> {
        // Safety net: refuse to operate on absurdly large inputs even
        // though the WS layer caps frames at MAX_WS_MESSAGE_BYTES. A
        // future change that raises that cap should be caught here too.
        debug_assert!(input.len() <= STATE_INPUT_HARD_CAP);

        let mut out = Vec::with_capacity(input.len());
        for &b in input {
            self.step(b, &mut out);
        }
        out
    }

    fn step(&mut self, b: u8, out: &mut Vec<u8>) {
        // Single-byte (8-bit) introducers — equivalent to ESC + intro byte
        // in 7-bit form. xterm.js emits 7-bit only, but a malicious client
        // could send the 8-bit form to bypass the ESC-prefix branch.
        // TM-INPUT.1/2/3 covers both forms.
        if self.state == State::Normal {
            match b {
                0x9B => { /* CSI single-byte — let through; CSI isn't in scope */ out.push(b); return; }
                0x9D => { self.osc_type_digits.clear(); self.state = State::OscPrefix; return; }
                0x90 | 0x9E | 0x9F | 0x98 => { self.state = State::StringDrop; return; }
                _ => {}
            }
        }

        match self.state {
            State::Normal => {
                if b == 0x1B {
                    self.state = State::SeenEsc;
                } else {
                    out.push(b);
                }
            }
            State::SeenEsc => match b {
                b']' => {
                    self.osc_type_digits.clear();
                    self.state = State::OscPrefix;
                }
                b'P' | b'_' | b'^' | b'X' => {
                    // TM-INPUT.3: DCS / APC / PM / SOS — drop until ST.
                    self.state = State::StringDrop;
                }
                _ => {
                    // Anything else after ESC (CSI introducer `[`, single-
                    // char functions, charset designators…). Re-emit
                    // verbatim — non-OSC/DCS/APC/PM/SOS is out of scope
                    // for this filter and the threat model.
                    out.push(0x1B);
                    out.push(b);
                    self.state = State::Normal;
                }
            },
            State::OscPrefix => {
                if Self::is_osc_terminator(b) {
                    // OSC ended before ';' — empty body, type = digits.
                    self.finish_osc_or_skip(out, Some(b));
                } else if b == b';' {
                    self.commit_osc_type(out);
                } else if b == 0x1B {
                    // ST start — ESC \
                    self.state = State::OscPrefixEscFallback();
                } else if b.is_ascii_digit() {
                    if self.osc_type_digits.len() >= OSC_TYPE_DIGITS_CAP {
                        // Pathologically long type id — drop the sequence.
                        self.osc_type_digits.clear();
                        self.state = State::OscDrop;
                    } else {
                        self.osc_type_digits.push(b);
                    }
                } else {
                    // Non-digit, non-; — malformed OSC. Drop to be safe.
                    self.osc_type_digits.clear();
                    self.state = State::OscDrop;
                }
            }
            State::OscAllow => {
                if Self::is_osc_terminator(b) {
                    out.push(b);
                    self.state = State::Normal;
                } else if b == 0x1B {
                    self.state = State::OscAllowEsc;
                } else {
                    out.push(b);
                }
            }
            State::OscAllowEsc => {
                if b == b'\\' {
                    // ST = ESC \ — emit both bytes, end sequence.
                    out.push(0x1B);
                    out.push(b'\\');
                    self.state = State::Normal;
                } else {
                    // Stray ESC inside an allowed OSC — bail by emitting
                    // ESC and treating the next byte as Normal.
                    out.push(0x1B);
                    out.push(b);
                    self.state = State::Normal;
                }
            }
            State::OscDrop => {
                if Self::is_osc_terminator(b) {
                    self.state = State::Normal;
                } else if b == 0x1B {
                    self.state = State::OscDropEsc;
                }
                // Otherwise: drop silently.
            }
            State::OscDropEsc => {
                if b == b'\\' {
                    self.state = State::Normal;
                } else {
                    // Stray ESC, continue dropping.
                    self.state = State::OscDrop;
                }
            }
            State::StringDrop => {
                if Self::is_osc_terminator(b) {
                    self.state = State::Normal;
                } else if b == 0x1B {
                    self.state = State::StringDropEsc;
                }
            }
            State::StringDropEsc => {
                if b == b'\\' {
                    self.state = State::Normal;
                } else {
                    self.state = State::StringDrop;
                }
            }
        }
    }

    /// OSC sequences terminate with BEL (0x07). ESC \ is handled
    /// explicitly in the *Esc states above.
    #[inline]
    fn is_osc_terminator(b: u8) -> bool {
        b == 0x07
    }

    /// `;` seen in OscPrefix: type is known. Decide drop/allow and
    /// transition. On allow, emit the prefix we held so far so downstream
    /// sees a well-formed OSC.
    fn commit_osc_type(&mut self, out: &mut Vec<u8>) {
        let drop = matches!(
            std::str::from_utf8(&self.osc_type_digits)
                .ok()
                .and_then(|s| s.parse::<u32>().ok()),
            Some(52) | Some(8),
        );
        if drop {
            self.osc_type_digits.clear();
            self.state = State::OscDrop;
        } else {
            out.push(0x1B);
            out.push(b']');
            out.extend_from_slice(&self.osc_type_digits);
            out.push(b';');
            self.osc_type_digits.clear();
            self.state = State::OscAllow;
        }
    }

    /// OSC terminator hit while still in prefix (empty body). Same
    /// drop/allow decision; emit appropriately.
    fn finish_osc_or_skip(&mut self, out: &mut Vec<u8>, terminator: Option<u8>) {
        let drop = matches!(
            std::str::from_utf8(&self.osc_type_digits)
                .ok()
                .and_then(|s| s.parse::<u32>().ok()),
            Some(52) | Some(8),
        );
        if !drop {
            out.push(0x1B);
            out.push(b']');
            out.extend_from_slice(&self.osc_type_digits);
            if let Some(t) = terminator { out.push(t); }
        }
        self.osc_type_digits.clear();
        self.state = State::Normal;
    }
}
```

Notes on the sketch:

- The `State::OscPrefixEscFallback()` placeholder above will collapse into
  a small additional Esc state in the final implementation — kept verbose
  in the sketch to make the decision tree explicit.
- The filter touches the in-flight byte stream once. The per-byte branch
  count is small (constant-time) and bound by the input length.

### 3.3 `crates/claude-phone-gateway/src/routes/phone_ws.rs`

Wire the filter into the outgoing task. The filter is **per-connection**
(constructed inside `handle_socket`), so two phones cannot contaminate
each other's state.

```rust
async fn handle_socket(mut socket: WebSocket, state: PhoneWsState, token_str: String) {
    // … unchanged setup …

    let (mut sink, mut stream) = socket.split();
    let mut rx_from_wrapper = handle.rx;
    let to_wrapper = handle.session.to_wrapper.clone();
    let cancel = handle.session.cancel.clone();

    // TM-INPUT.1/2/3: stateful ANSI filter on phone→wrapper binary frames.
    // Strips OSC 52 / OSC 8, drops DCS / APC / PM / SOS; OSC 0/1/2 and
    // CSI/charset-designator sequences pass through. Text frames are JSON
    // (parsed by serde_json which carries its own 128-depth recursion
    // limit, TM-INPUT.5) — they bypass the byte filter.
    let mut input_filter = crate::input_filter::AnsiInputFilter::new();

    let cancel_outgoing = cancel.clone();
    let outgoing_task = tokio::spawn(async move {
        loop {
            let cancelled = cancel_outgoing.cancelled();
            tokio::pin!(cancelled);
            tokio::select! {
                biased;
                _ = &mut cancelled => break,
                msg = stream.next() => {
                    let Some(Ok(msg)) = msg else { break };
                    let frame = match msg {
                        Message::Binary(b) => {
                            let filtered = input_filter.push(&b);
                            if filtered.is_empty() {
                                // The entire frame was inside a banned
                                // sequence. Do not forward an empty frame —
                                // wrapper has no use for it.
                                continue;
                            }
                            Frame::Binary(filtered)
                        }
                        Message::Text(t) => Frame::Text(t),
                        Message::Close(_) => break,
                        _ => continue,
                    };
                    if to_wrapper.send(frame).await.is_err() { break; }
                }
            }
        }
    });

    // … incoming_task and join unchanged …
}
```

Lines changed: ~10 (insertion only — the existing match block grows by
one branch, no removals).

```rust
// TM-INPUT.7: the only accepted bytes are ASCII alphanumerics plus `-`
// and `_`. Control characters (NUL, BEL, ESC, DEL, etc.), whitespace,
// path separators (`/`, `\`), URL-significant punctuation (`?`, `#`,
```

### 3.8 `crates/claude-phone-wrapper/src/cli.rs`

Add a validation function and call it in `Cli::parse_validated()`:

```rust
use std::path::PathBuf;

use clap::Parser;

#[derive(Parser, Debug)]
#[command(version, about = "Wrap `claude` with phone bridging support")]
pub struct Cli {
    /// Path to wrapper config TOML.
    #[arg(short, long, env = "CLAUDE_PHONE_WRAPPER_CONFIG")]
    pub config: Option<PathBuf>,

    /// Override the `claude` binary path.
    #[arg(long, env = "CLAUDE_PHONE_CLAUDE_BIN", default_value = "claude")]
    pub claude_bin: String,

    /// All remaining args are forwarded to `claude` as-is.
    #[arg(trailing_var_arg = true, allow_hyphen_values = true)]
    pub claude_args: Vec<String>,
}

#[derive(Debug, thiserror::Error)]
pub enum CliError {
    #[error("--claude-bin is empty; refusing to spawn nothing")]
    ClaudeBinEmpty,
    #[error("--claude-bin contains a control character at byte {0}; refusing to spawn")]
    ClaudeBinControl(usize),
}

impl Cli {
    /// Parse and validate. Failures abort the process loudly — `claude_bin`
    /// is fed to a process spawn, and silently re-shaping the string could
    /// pivot an injected env var into a different binary.
    ///
    /// TM-INPUT.8: control characters in `--claude-bin` are rejected at
    /// startup. CommandBuilder does NOT pass through a shell (`execve`
    /// directly), so meta-characters like `;` `|` `&` `$` are non-issues —
    /// they would only land if the user explicitly typed them. The real
    /// concern is environment-tampered `CLAUDE_PHONE_CLAUDE_BIN` containing
    /// e.g. embedded newlines that confuse log readers or NUL bytes that
    /// truncate the spawn target on some platforms.
    pub fn parse_validated() -> Result<Self, CliError> {
        let cli = Self::parse();
        cli.validate()?;
        Ok(cli)
    }

    fn validate(&self) -> Result<(), CliError> {
        if self.claude_bin.is_empty() {
            return Err(CliError::ClaudeBinEmpty);
        }
        for (i, b) in self.claude_bin.bytes().enumerate() {
            // Reject all C0 control chars (0x00-0x1F) and DEL (0x7F).
            // Tab/newline/CR have no legitimate place in a binary path.
            if b < 0x20 || b == 0x7F {
                return Err(CliError::ClaudeBinControl(i));
            }
        }
        Ok(())
    }
}
```

### 3.9 `web/src/components/Terminal/useTerminal.ts`

Register no-op OSC handlers for 52 and 8 immediately after `term.open(...)`
and audit the existing parser surface. xterm.js's
`term.parser.registerOscHandler(id, handler)` registers a callback;
returning `true` tells xterm.js the handler consumed the sequence so the
default action is suppressed.

```typescript
// … existing setup through term.open(containerRef.current) …

// TM-INPUT.4: explicitly no-op OSC 52 (clipboard write) and OSC 8
// (hyperlinks). Gateway phone→wrapper path already strips these
// (TM-INPUT.1/2), but wrapper→phone is unfiltered by design — claude
// content is rendered as-is. If claude ever proxies adversarial bytes
// (e.g. via a tool result that contains terminal escapes), these
// handlers stop xterm.js from acting on them.
//
// Returning `true` from an OSC handler tells xterm.js the sequence was
```

## 6. Risk-of-regression notes

### 6.1 Anything that touches `phone_ws::handle_socket`

The filter is **moved-in to the spawned task**; the outer function returns
the same `Response` shape. Any existing test that drives a phone WS
flow (e.g. `e2e_test.rs`) must keep passing. The forward-looking integration
test in commit 2 confirms plain ASCII traffic flows unchanged.

### 6.2 Filter stateful across frames — task lifetime invariant

The filter is owned by the outgoing task closure. If a future refactor
moves the binary-frame handling outside the spawned task (e.g. into a
tower layer), the per-connection state must move with it. The
forward-looking test #11 (OSC 52 split across two frames) catches a
regression where the filter is accidentally re-created per frame.

### 6.3 JSON depth — `unbounded_depth` feature

Tests #30/#31 will fail if anyone enables serde_json's `unbounded_depth`
feature (which is opt-in via the `serde_json/unbounded_depth` cargo
feature). Sub-spec 4.14 adds a `cargo-deny` bans rule for it; this
sub-spec carries the runtime test.

### 6.4 ServeDir behavior depends on tower-http version

Tests #19-#21 assert the **current** tower-http 0.6 behavior. If
tower-http changes path normalization (unlikely on a stable line, but
possible across major versions), the tests will fail loudly — that's
the point. A failure means "review the new behavior, decide if it's
still TM-INPUT.6-compliant, update the test or upgrade with care".

### 6.5 Wrapper CLI validation — environment-tampered env var

The validation runs on the parsed `claude_bin` regardless of whether the
value came from `--claude-bin`, the env var, or the default. So an
environment-tampered `CLAUDE_PHONE_CLAUDE_BIN` with control chars is
rejected at startup. Tests #33-#36 cover this; the env-var path is
exercised by clap's `env` attribute (no extra test needed for the
mechanism — clap is well-tested upstream).

### 6.6 xterm.js handler registration timing

OSC handlers are registered **after** `term.open(...)` per the xterm.js
docs (parser is initialized lazily on open). If a future refactor moves
`term.open` later (e.g. behind a useEffect dependency change), the
handlers must follow. Tests #39/#40 confirm registration takes effect.

### 6.7 Forward-looking guards (do NOT remove without re-design)

- `// TM-INPUT.1/2/3:` comment block above `AnsiInputFilter::new()` call
  in `phone_ws.rs`.
- `// TM-INPUT.5:` module doc in `protocol.rs`.
- `// TM-INPUT.6:` comment at `http.rs:74` ServeDir line.
- `// TM-INPUT.7:` comments in `token.rs::is_base64url_byte` and
  `phone_ws.rs:38`.
- `// TM-INPUT.8:` comment in `cli.rs::validate`.
- `// TM-INPUT.4:` comment in `useTerminal.ts` registerOscHandler block.
- Tests #4, #5, #6, #7, #8, #9, #10, #11, #12, #14, #17, #18, #19-#21,
  #23-#27, #28, #29, #30, #31, #33-#36, #39, #40 are invariant tests, not
  behaviour tests.
